해당 서버로 데이터 파일을 옮겨야 함

모델 예측 전산 작업은 데이터 전처리 -> 모델 예측 -> 중요변수 추출 -> 상관계수 계산 으로 진행

(이후 결과 파일 집계, 정리 / 전망보고서 작성 / 플랫폼 데이터 업로드는 Tech VDI, Local PC 환경에서 진행함)

데이터 전처리 단계

1. 업로드한 엑셀/텍스트 파일을 읽어서 모델에서 쓸 수 있는 형태의 데이터로 변환
2. LSEG 데이터스트림을 통해서 그 외 데이터 확보, 모델에서 쓸 수 있는 형태의 데이터로 변환
3. 브렌트, 두바이유의 시가, 고가, 저가, 종가 등으로 기술적/통계석 분석 변수 생성
4. 앞에서 작업된 데이터셋을 데이터셋 성격에 맞춰서 그룹핑을 하고 기존 데이터셋에 추가함
5. 모델에 입력하기 위한 데이터셋 생성 작업 진행(입수 지연 반영, 모델에서 사용하지 않는 변수 제거 등..)

모델 예측

1. 딥러닝 모델: 두바이/브렌트유 예측 모델을 각기 수행
2. 머신러닝 모델: 두바이/브렌트유 예측 모델을 각기 수행

중요 변수 추출

1. 딥러닝/머신러닝 모델에서 뽑는 중요변수 형태가 다른데 형태를 맞추는 작업을 함
2. 모델에서 뽑은 중요 변수를 일, 주, 월별로 계산
3. 전망리포트에 쓰일 중요 변수 리스트, 최근 데이터 추이 등을 추가 작업

상관계수 계산

1. 모델에서 사용한 변수를 기반으로 현재까지 1,3,5년으로 상관계수를 계산함

---

이 단계를 마치면 모델 예측 결과가 나옴

1. 위의 수치를 기반으로 집계, 결과 파일 생성
2. 중요 변수 리스트를 엑셀로 작성
3. 전망 리포트 작성

전망 리포트 작성

1. 리포트는 3단계로 나눠서 작성함
   - 1단계는 브렌트유 가격, 모델에서 활용하는 최근 데이터 변수를 기반으로 
     향후 1주일 브렌트유 가격 전망을 LLM이 2개의 상반되는 입장(상승, 하락)을 토론하고 최종 결론을 냄
   - 2단계는 1단계 결과물(상승, 하락 근거, 토론 결과), 외부 정보 수집을 기반으로 전망 보고서를 쓰기 위한 기초 자료를 작성함
   - 3단계는 2단계에서 작성한 파일과 리포트 형식에 대해 정의한 자료를 기반으로 전망리포트를 작성함

```

STAGE_NOTE = "이후 2단계에서 이 1개월 View를 향후 1주일 단기 시나리오로 압축·재해석합니다."
```

##### DATE_RANGE 자동 생성 규칙

- `news_report`의 관측 범위는 **오늘 기준 최근 30일**로 설정한다.
- 표기 형식은 반드시 아래를 따른다.

```
최근 30일(YYYY-MM-DD~YYYY-MM-DD)
```

- 정의
  - 종료일(End): 오늘 날짜
  - 시작일(Start): 오늘 날짜에서 30일 전
- 출력 및 기준 서술 시 반드시 위 형식의 DATE_RANGE를 생성해 사용한다.

---

##### [DERIVED RULES – 자동 설정 (수정 금지)]

```
COUNTER_SCENARIO = "하락" if MAIN_SCENARIO == "상승" else "상승"
MAIN_HISTORY = "bull_history" if MAIN_SCENARIO == "상승" else "bear_history"
COUNTER_HISTORY = "bear_history" if MAIN_SCENARIO == "상승" else "bull_history"
MAIN_LABEL_EN = "Bull(상승)" if MAIN_SCENARIO == "상승" else "Bear(하락)"
OUTPUT_TITLE = f"브렌트유 ‘{MAIN_SCENARIO}’ 전망 리포트"
LEAD_SENTENCE = f"{HORIZON} 기준 브렌트유는 완만한 {MAIN_SCENARIO} 가능성이 우세합니다."
BALANCE_SENTENCE = f"다만 {COUNTER_SCENARIO} 요인도 존재하며, 단기 반대 시나리오가 일부 구간에서 나타날 가능성도 배제하기 어렵습니다."
```

##### [GLOBAL STABILITY RULES – 1단계 흔들림 최소화 (필수)]

##### 1) 사실·수치·사건 생성 금지

- 입력 자료에 없는 사실/숫자/사건을 새로 만들지 않는다.

```
침공 / 체포 / 구금 / 봉쇄 / 공격 / 정권붕괴 / 쿠데타 / 제재 / 시설피격 / 수출차질
```

- 확인되지 않으면 반드시 아래 문장으로만 처리한다.

```

```

##### 2) 메커니즘 우선 원칙

- 모든 이벤트·정책·지표는
   **“왜 유가에 영향을 주는지”를 1~2문장 메커니즘**으로 연결한다.
- 지정학 리스크는 반드시 아래로 구분한다.

```

```

##### 3) 출처·각주 규칙

- 작업 3(1-C/1-D)에서 **확인된 외부 근거**에만 각주 [1][2]…를 부착한다.

   **각주·참고문헌 추가 금지**.

##### 5) 형식 고정

##### [HIGH-IMPACT COVERAGE CHECK – 고충격 이벤트 누락 방지 (보강형)]

##### 1단계 필수 점검 범주

작업 3(1-C/1-D) 작성 전, 아래 항목이 **명시적으로 존재하는지** 먼저 확인한다.

**B. 해상 물류·초크포인트** 홍해 / 수에즈 / 호르무즈 / 항로 차질 / 보험료·운임 급등

**C. 소비국 정책** SPR / 제재 강화·완화 / 대외정책 변화

**D. 국제기구 전망** IEA / EIA / OPEC 수급 전망 변경(공급·수요·재고)

##### [추가 증거 수집 – High-Impact Sweep (필수)]

news_report에 A~D가 없거나 일부만 있는 경우, 아래 절차를 수행한다.

**검색 범위**

- 기본: DATE_RANGE(최근 30일)
- 국제기구 전망은 발표 주기 고려 → **최근 3개월까지 확장 허용**

**출처 우선순위**

1. Reuters
2. Bloomberg / WSJ / FT
3. IEA / EIA / OPEC / 정부·공식기관
4. Investing.com (보조 확인용)
5. Wikipedia (정의·기관 설명만 허용, 사건 근거 사용 금지)

**결과 처리**

- 신뢰 출처에서 사실 확인 시
   → 본문 반영 + 각주 부착
- 출처 간 상충·불명확
   → 확인된 범위까지만 중립 서술
- 끝까지 확인되지 않으면
   → `"허용된 출처 범위에서 확인되지 않습니다."` 로만 처리

##### **[3] GLOBAL RULES (운영 핵심 규칙)**

###### **3-1) Lexicon 최우선**

- oil_market_lexicon_v3.json의 report_layout/heading_rules/sentence_rules/analysis_requirements/wording_preferences/prohibited_patterns/reference_style를 최우선 준수.

- 단, **외부 근거 수집/레퍼런스 검증/고충격 이벤트 처리/주간 압축 및 ‘4주 흐름→1주 결론’ 규칙은 본 프롬프트 우선**.

###### **3-2) “1개월 관측 → 1주일 결론” 강제**

- DATE_RANGE는 **근거 탐색·상황 설명(배경)**에만 사용.

- 최종 결론·시사점·프레이밍은 **반드시 REPORT_HORIZON(향후 1주일)** 기준으로 작성.

- 서론(1장)과 결론(4장)에 **“향후 1주일 브렌트유 시장”** 문구를 반드시 포함.

- 서론에서 “최근 1개월 관측 구간”은 **‘배경’으로만** 두고, 본문 전개는 “향후 1주일”을 중심으로 연결한다.

###### **3-3) Stage1_output “주간 압축” 규칙(★핵심)**

- stage1_output의 주차 테이블([1-B])에서 **가장 최근 주차(=표의 마지막/최신 구간)**를 최우선 앵커로 삼아 1주일 전망 근거를 구성.

- 그 이전 2~3주는 “추세/배경”으로만 요약하되, 최종보고서 톤처럼 **4주 흐름을 ‘급등–확대–되돌림–불확실성 유지’처럼 연결 서사로 묶어** 시장 개요에 먼저 배치한다.

- **주간 압축 서술은 ‘1문단 안에서 2~4문장’으로 요약**하되,
  
  - (a) 변동률 5% 초과 항목(또는 stage1_output에서 강조된 항목)만 수치로 제시
  
  - (b) 나머지는 정성 표현으로 처리
  
  - (c) “물리(물류/공급망) vs 심리(리스크 프리미엄/변동성)” 구도를 최소 1회 포함

- stage1_output에 “…” 또는 생략 표시가 있는 값은 **외부 사실/수치로 보완하지 말고**, “입력자료 생략으로 수치 확인 제한”이라고만 처리.

###### **3-4) 사실/수치 생성 금지**

- 입력(stage1_output, 외부 출처) 없는 사실/숫자/사건을 만들지 말 것.

- 특히 “침공/체포/구금/봉쇄/공격” 등 고충격 단어는 **검증 통과 출처로 확인된 경우에만** 사용.

###### **3-5) 트레이딩/기술적 지표 금지**

- “매수/매도/롱/쇼트/포지션/손절/전략” 전면 금지.

- RSI/MACD/EMA/SMA/ATR 등 기술적 지표 언급 금지.

###### **3-6) 본문 형식 고정(1~4장)**

- 1~4장 본문은 **문단형 서술만 허용**.

- 불릿/번호/“첫째·둘째·셋째” 포함한 리스트형 표현 금지.

- 대신 “또한/한편/동시에/결과적으로/종합하면/다만/특히/그러나” 등 연결어로 확장.

- 문체는 보고서형(단정·명료)으로 유지하되, 최종보고서처럼 **문맥을 풍부하게 연결**한다(“~로 정리된다”, “~로 해석된다”, “~을 시사한다”).

###### **3-7) ‘품질체크/한계’ 문단 반영(최종보고서 스타일 반영)**

- internal_notes 또는 stage1_output에 QC/제한사항(예: 변수 집중, 뉴스 관측 기간 불일치, 펀더멘털 동시 점검 제한)이 있으면,
  
  - 4장(시장 해석 및 시사점)에서 **1문단 이상**으로 “내부 일관성은 양호하나 한계가 있다”는 형태로 반영한다.

- 단, **새로운 결함/한계를 만들어내지 말고** 입력에 있는 내용만 사용한다.

##### **[5] HIGH-IMPACT EVENT 반영(검증 통과 시만)**

###### **5-1) 후보 탐지 쿼리(필수)**

- “Venezuela oil” + (invasion OR arrest OR coup OR sanctions OR exports)

- “sanctions tanker shadow fleet”

- “pipeline outage” OR “port closure” OR “force majeure” + oil

###### **5-2) 검증 게이트(강제)**

이벤트를 본문에 포함하려면 아래 모두 통과:

- 화이트리스트 도메인 기사/보고서 **본문 페이지** 접근 가능

- **보도 날짜 또는 발생 시점(YYYY-MM-DD)** 명시

- 본문에서 인용할 핵심 주장(수출 차질/제재 발표 등)이 기사 내용으로 확인됨

- 단정이 강해질 경우 **추가 1개 출처로 교차 확인 우선**

검증 실패 시 본문에는 **“현재 허용 출처 범위에서 확인되지 않습니다.”** 1문장만 허용(구체 팩트 서술 금지).

###### **5-3) 서술 방식(강제)**

- 포함된 고충격 이벤트는 2-3(지정학)에서 최소 1문단 이상 설명.

- 반드시 “공급 차질(물리)” vs “리스크 프리미엄(심리)”를 분리해 메커니즘 중심으로 서술.

- 공급/수출 직결이면 2-1(수급)에도 연결 문장 1개 이상 추가.
